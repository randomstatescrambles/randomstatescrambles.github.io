About Scramble GJenerator

Scramble GJenerator is a web-based random state scramble service. Writing a random state Rubik's cube scramble generator is hard. Fortunately, this has already been done, so I didn't have to. However, if someone wanted to create a mobile app timer, or a browser based javascript timer, they wouldn't be able to use the tnoodle command line program. They would have to either figure out how to create their own random state scramblers in the language they are developing in, or use random move scramblers. Scramble GJenerator solves this problem by supplying scrambles through a server client model. All you have to do is ask the Scramble GJenerator server for a random state scramble, and the server will supply you with one, generated straight from TNoodle.

Why random state scrambles?

For random state scrambles, a random state of a puzzle is chosen, and then a sequence of moves is generated to reach that state. This guarantees that every competiton legal puzzle state has an equal chance of occuring. Any sequence of n random moves will tend toward the set of scrambles that are n moves away from solved. Random state scrambles guarantee that there is no statistical bias toward easy solutions, and therefore are much better for practice.
